程序猿的时间观

2014.08.13

昨天不知道在什么情况下自言自语问了同事一句：今天周几？周二了吗？这周过得真快啊！

如果是以前，周二对于我来说一个星期才过两天，离周末还远着，就算不盼着周末，也不会说在周二就觉得这一周就要快过完的感觉。

当我说完「这周过得真快」后，内心不知为何开始着急、焦虑。出来工作三年多了，对于程序员的工作的量化评估逐渐准确起来；在IT行业涉足越深，也逐渐有了一些自己对这行业的想法，想要去做一些自己想要做的事；还有来自上一代人对自己婚姻的催促，忽然觉得有好多事等着自己去做啊！而我总是想着这些事能不能这周内搞定啊，周三搞定这个、周四搞定那个、周五再搞定那个，这样的任务安排能否顺利执行？要是这周的时间再多一点就好了。

也许正因为有了这样的想法，才会有这种时间过得很快的错觉，难道这就是奔三穷屌丝的焦虑？

而我再仔细分析自己的时间分配，对于工作量的评估，作为程序猿加班是必然了。而业余时间的选择，一个是做 爱做的事，通过自己的才能和专业优势更好地在人生事业发上展自己；另一个是做爱 做的事，也就是找个女朋友那啥，给家里人一个交代，更重要的是给自己一个交代。这两者可以同时进行，或者先专心搞好一个再专心搞另一个。那我这里就再分三种情况讨论。

一、两者同时进行。这种情况下，加班晚了回来是一边搞事业一边搞女人，说得好像很刺激的样子，想要找到平衡点两边都不冷落是一个严峻的考验。主要是上班写代码回来要保持手热的状态，回来继续搞的话，有种屌丝注定孤独一生的感觉；如果是回来赏花赏月赏秋香，搞事业的状态就完全没有了啊；如果是让她坐在大腿上手把手一起搞事业呢？那画面太美我不敢看。当然，程序员的思维一旦养成，if { do this }， else { do that }，在现实中你也不可能用多线程，如果不照这样的程序走，这种两者同时进行的话某些地方没处理好总感觉容易出bug。

二、先专心解决终生幸福大事。这种情况下，在现实看来，这是非常理想的状态，上班累了，回来哈皮，过个一两年，再带个娃，生活乐此不疲啊。生活安定了，开始事业上发力，走向人生巅峰。微信鸡汤上有云：人生最好的投资不是房、不是金、不是公司，而是一个对的女人。「我一生最好的投资就是选对了老婆。」——巴菲特。问题来了，一个屌丝猿如何投资一个对的女人？在猿的周围全是雄性动物的情况能否找到这样的女性？通过其他方式相识而来的优秀女性能否对一个整天加班的猿有一个客观正确的评价？我如果是巴菲特，想投资多少好女人就投资多少好女人，而且是只谈情不上床的那种行不行？这种情况对于程序猿来说有个弊端，因为程序猿就像妓女，靠青春吃饭，安逸了两三年再想通过自己的特长去搞事业（尤其是瞬息万变的IT行业）多少有点难。好的话或许会得到自己想要的结果，不好的话也就带带娃了此残生了。

三、先专心事业。大丈夫行走天下，何患无妻？作为一个有理想有追求的热血青年，能有什么比得上挥洒拼搏的汗水所带来的喜悦。我所指的事业可以是自己个人的事业，也可以是把公司的事业当自己的事业。无论是哪种事业，无非都是希望自己有更好的人生价值体现，同时又能满足优质生活和提升逼格的愿景。当然，专心搞事业也是有风险的，就像你懂得很多道理，但还是过不好这一生一样，搞个几年没搞好，人又老了，到时候更难找到合适的另一半，只能随随便便找个人凑合着一起了此残生了。

从上面三种情况来看，可以有一个共同的结论：只要你牛逼，怎么做都行；不牛逼，老老实实了此残生（我去，总结出来后发现上面写的这么几大段都是废话）。

这也得出了一个让我惭愧的结论：原来我的焦虑是因为我还不够牛逼，小扎和我这么大的时候，他的脸书都上市了，而我现在是恨不得一天能当一年用，来弥补我和某些人的差距，我的修行还不够啊。

程序猿的修炼是无止境的，因此作为一个猿，对时间的掌控更是要分毫拿捏的好，这时候我觉得又可以分三种情况来分析每天的时间如何分配自身修炼和自身幸福的问题：1.两者同时进行；2.先解决终生幸福；3.先专心修炼。（我好像写了个递归，而且没有终止条件）
